# Radio-86RK programmer's reference

A quick reference for writing or reversing Intel 8080 code for the
Radio-86RK. For the monitor jump table see [MONITOR.md](./MONITOR.md).

Source of truth for everything on this page is the monitor ROM at
[begoon/rk86-monitor](https://github.com/begoon/rk86-monitor/blob/main/monitor.asm).

## Memory map (32 KB RAM machine)

| Range       | Contents                                   |
|-------------|--------------------------------------------|
| 0000h–75FFh | Free user RAM (30 KB)                      |
| 7600h–765Fh | Monitor variables (see below)              |
| 7660h–76CFh | Monitor stack (grows down from 76CFh)      |
| 76D0h–7FF4h | Video memory — 30 rows × 78 cols = 2340 B  |
| 8000h–8003h | PPI D20 (keyboard) — К580ВВ55              |
| A000h–A003h | PPI D14 (tape / ROM-disk) — К580ВВ55       |
| C000h–C001h | CRT controller — К580ВГ75                  |
| E004h–E008h | DMA — К580ВТ57                             |
| F000h       | ROM extension slot (aliases to F800h)      |
| F800h–FFFFh | Monitor ROM (2 KB)                         |

`memory_top = 75FFh` is the documented upper bound of free user memory;
programs typically start with `lxi sp, 75FFh` (or any lower address).
The monitor itself runs with `SP = 76CFh`.

## Peripheral registers

### Keyboard (К580ВВ55 "D20" at 8000h)

| Addr  | Role                                                  |
|-------|-------------------------------------------------------|
| 8000h | Port A — row scan mask (output, active low)           |
| 8001h | Port B — column data (input, active low)              |
| 8002h | Port C — modifier keys + tape I/O                     |
| 8003h | Control word (monitor programs `8Ah` at cold start)   |

Port C bits: bit 5 = SHIFT (СС), bit 6 = CTRL (УС),
bit 7 = РУС/ЛАТ toggle, bits 0–3 = tape in/out.

Matrix scan idiom:

```asm
    mvi  a, 0FDh        ; select row 1 (0 = active)
    sta  8000h
    lda  8001h          ; read columns
    cma                 ; 1 = pressed
```

Monitor callers should prefer `getc` / `kbhit` / `scan_kbd`
(see [MONITOR.md](./MONITOR.md)) over raw matrix reads.

### CRT controller (К580ВГ75 at C000h)

| Addr  | Role                                   |
|-------|----------------------------------------|
| C000h | Parameter / data register              |
| C001h | Command register (W) / status (R)      |

After a reset the screen is blank until the CRT is reprogrammed.
Call `F82Dh` (monitor `video`) to reinitialize it — this is what the
cold-start path does.

### Tape / ROM-disk PPI (К580ВВ55 "D14" at A000h)

| Addr  | Role                            |
|-------|---------------------------------|
| A000h | Port A — ROM-disk data          |
| A001h | Port B — ROM-disk addr low     |
| A002h | Port C — ROM-disk addr high    |
| A003h | Control word                    |

Most programs use the monitor's `inpb` / `outb` / `inpblock` /
`outblock` entries instead of touching this PPI directly.

| 7604h | `ap2_seq`         | 1    | Byte index in an АР2 escape sequence       |
| 7605h | `pressed_key`     | 1    | FFh = key held, 00h = released             |
| 7606h | `ruslat_flag`     | 1    | 00h = LAT, FFh = RUS                       |
| 7609h | `keyboard_repeat` | 2    | L = key code, H = autorepeat counter       |
| 760Dh | `temp_sp`         | 2    | Temp SP slot used by monitor commands      |
| 7614h | `pc_storage`      | 2    | Saved PC (G-command / breakpoint)          |
| 7616h | `hl_storage`      | 2    | Saved HL                                   |
| 7618h | `bc_storage`      | 2    | Saved BC                                   |
| 761Ah | `de_storage`      | 2    | Saved DE                                   |
| 761Ch | `sp_storage`      | 2    | Saved SP                                   |
| 761Eh | `af_storage`      | 2    | Saved A+PSW                                |
| 7623h | `stop_addr`       | 2    | Breakpoint address                         |

| 7629h | `cmd_param_2`     | 2    | 2nd parameter                              |

| 7630h | `tape_write_const`| 1    | Default `1Dh` (≈406 µs / bit)              |

The `cursor_addr` at 7600h is handy for direct-to-video tricks (write
bytes to `(HL)` after `lhld 7600h`); `putc` updates it for you.

## Video memory

```
rows 0..29  ×  78 columns per row  =  2340 bytes at 76D0h

video address(y, x) = 76D0h + y*78 + x
```

Each byte is a **character code**, not a pixel pattern — the ВГ75
fetches the glyph from the character-generator ROM. The default text
window used by the monitor covers **columns 8..71, rows 3..27**
(64 × 25). Its top-left is at `77C2h = 76D0h + 3*78 + 8`.

## Character set

One byte per cell, KOI-7-style:

| Range     | Contents                                                 |
|-----------|----------------------------------------------------------|
| 00h–1Fh   | Block/line graphics and arrows (many cells render blank) |
| 20h–5Fh   | Standard ASCII                                           |
| 60h–7Fh   | Cyrillic uppercase (Latin → Russian mapping below)       |
| 80h–FFh   | Inverse video of 00h–7Fh                                 |

Cyrillic row (60h–7Fh) as typed from a Latin keyboard, the way it
appears in source strings:

```
` a b c d e f g h i j k l m n o p q r s t u v w x y z { | } ~
Ю А Б Ц Д Е Ф Г Х И Й К Л М Н О П Я Р С Т У Ж В Ь Ы З Ш Э Щ Ч
```

A fuller glyph table lives in
`https://github.com/begoon/lestnica-i8080/info/rk86-charmap.md` and in `https://github.com/begoon/rk86-font/` (bitmap dump of the 6×8 character ROM).

## Control codes (as seen by `putc`)

| Code | Effect                                          |
|------|-------------------------------------------------|
| 07h  | Beep (tone = B/C at routine entry)             |
| 08h  | Cursor left                                     |
| 0Ah  | Line feed                                       |
| 0Ch  | Home cursor                                     |
| 0Dh  | Carriage return                                 |
| 18h  | Cursor right                                    |
| 19h  | Cursor up                                       |
| 1Ah  | Cursor down                                     |
| 1Bh  | Start of escape sequence (see below)            |
| 1Fh  | Clear screen (and home the cursor)              |

Only one escape sequence is honored: `ESC Y y x`, where `y` and `x` are
the target row and column each **biased by +20h**. So to move to
row 5, column 10:

```asm
    mvi  c, 1Bh
    call 0F809h
    mvi  c, 'Y'
    call 0F809h
    mvi  c, 5 + 20h
    call 0F809h
    mvi  c, 10 + 20h
    call 0F809h
```

## Keyboard

`getc` at `F803h` blocks until a key is pressed and returns one of:

| Code       | Key                                                   |
|------------|-------------------------------------------------------|
| 00h        | Ctrl-@ (NUL) — used as `F4` in some programs          |
| 01h, 02h   | Ctrl-A / Ctrl-B — `F5` / extra function-key slots     |
| 03h        | Ctrl-C — monitor programs treat it as "reset"         |
| 08h        | Cursor left / BS                                      |
| 09h        | Tab                                                   |
| 0Ah / 0Dh  | LF / CR (Enter)                                       |
| 18h        | Cursor right                                          |
| 19h        | Cursor up                                             |
| 1Ah        | Cursor down                                           |
| 20h–7Fh    | Printable character (Latin or Cyrillic per РУС/ЛАТ)   |

F1..F5 land on 00h..04h in most keymaps. `kbhit` at `F812h` is the
non-blocking variant — returns 00h if nothing is pressed, otherwise the
key code. `scan_kbd` at `F81Bh` returns FFh for "no key", FEh for the
РУС/ЛАТ toggle alone, or the raw key code.

## Tape

Programs almost always go through the monitor:

```asm
    lxi  h, start
    lxi  d, end
    call 0F827h        ; outblock - write HL..DE, BC=checksum out
    ...
    lxi  h, 0          ; 0 = use address from the block header
    call 0F824h        ; inpblock - read, CY set on checksum error
```

Low-level constants worth knowing:

| Value | Meaning                                     |
|-------|---------------------------------------------|
| E6h   | Sync byte — also a literal marker in files  |
| 2Ah   | Default read timing constant (≈591 µs/bit)  |
| 1Dh   | Default write timing constant (≈406 µs/bit) |

## File formats

Address fields are 2-byte **big-endian**; `end_addr` is the address of
the **last** byte (inclusive, not one-past-the-end). `payload` is
`end_addr − start_addr + 1` bytes of raw i8080 code/data — identical
to what you'd get from `.bin`. `checksum` is the monitor's `chksum`
routine (`F82Ah`) run over `start_addr..end_addr`, stored big-endian.

| Extension     | Layout                                                                        |
|---------------|-------------------------------------------------------------------------------|
| `.bin`        | Raw payload only, no header or trailer.                                       |
| `.rk`, `.rkr` | `start_hi start_lo end_hi end_lo` ‖ `payload` ‖ `E6 sum_hi sum_lo`            |
| `.pki`, `.gam`| `E6 start_hi start_lo end_hi end_lo` ‖ `payload` ‖ `E6 sum_hi sum_lo`         |

So `.pki` / `.gam` and `.rk` / `.rkr` differ only in the leading sync
byte: `.pki` / `.gam` repeat the `E6h` sync byte before the address
header, `.rk` / `.rkr` omit it. The trailing `E6 sum_hi sum_lo` is the
same in both.

Examples — a 3-byte payload `21 00 00` (`lxi h, 0`) at address 3000h,
checksum `0021h`:

```
.bin   21 00 00
.rk    30 00 30 02 | 21 00 00 | E6 00 21
.pki   E6 30 00 30 02 | 21 00 00 | E6 00 21
```

This is exactly the `.rk` tape-file format the playground emits.

## Programming idioms

### Program skeleton

```asm
    org  3000h             ; any address in 0000h..75FFh
    lxi  sp, 75FFh         ; stack at the top of user RAM

    lxi  h, hello
    call 0F818h            ; puts
    jmp  0F86Ch            ; back to the monitor prompt

hello: db 'hi, world', 0Dh, 0Ah, 0
```

### Clear screen + print at (row, col)

```asm
    mvi  c, 1Fh            ; clear screen + home
    call 0F809h
    mvi  c, 1Bh            ; ESC
    call 0F809h
    mvi  c, 'Y'
    call 0F809h
    mvi  c, 3 + 20h        ; row 3
    call 0F809h
    mvi  c, 8 + 20h        ; col 8
    call 0F809h
    lxi  h, msg
    call 0F818h
```

A reusable `set_cursor` that patches an ESC-Y template in place
(preserves nothing by default) is how `docs/examples/dumped.asm`
does it.

### Read one key

```asm
wait:   call 0F803h         ; getc, blocks
        cpi  0Dh            ; Enter
        jz   done
        cpi  1Bh            ; ESC
        jz   abort
        ...
```

### Direct video write at current cursor

```asm
    lhld 7600h              ; HL = current cursor address
    mvi  m, 17h             ; 17h = full block
```

Direct writes bypass `putc`, so the monitor's own cursor tracking
stays in sync only if you follow up with a `putc` or an `ESC Y`
sequence.

### Exit back to the monitor

```asm
    jmp  0F86Ch             ; warm restart (keeps screen state)
    ; or
    jmp  0F800h             ; cold restart (reinitializes everything)
```

Pressing Ctrl-C during `getc` in a well-behaved program should route
to `0F800h` (the hard reset). The dump editor in
`docs/examples/dumped.asm` is an example of that pattern.

## Reversing cheat sheet

Patterns you will see over and over in RK86 binaries:

| Sequence                          | Meaning                        |
|-----------------------------------|--------------------------------|
| `31 FF 75`                        | `lxi sp, 75FFh` — stack init   |
| `CD 18 F8`                        | `call puts`                    |
| `CD 09 F8`                        | `call putc`                    |
| `CD 03 F8`                        | `call getc`                    |
| `CD 15 F8`                        | `call hexb` — print A as 2 hex |
| `C3 6C F8`                        | `jmp` to monitor prompt        |
| `0E 1F CD 09 F8`                  | clear screen                   |
| `0E 1B CD 09 F8 0E 59 CD 09 F8 …` | ESC-Y cursor positioning       |
| `2A 00 76`                        | `lhld 7600h` — cursor address  |
| `3A 06 76`                        | `lda 7606h` — РУС/ЛАТ flag     |
| Any `CPI 03h / JZ F800h`          | "Ctrl-C returns to monitor"    |

Control-code checks on `getc` results (`CPI 08h / 18h / 19h / 1Ah`)
almost always indicate cursor-key handling.

Self-modifying `MVI A, nn` followed by a `STA ??+1` elsewhere is a
common idiom for storing a tiny amount of state inline — the dump
editor uses this to remember "mid-byte edit in progress" without
spending a RAM byte.
